# Professional-Corporate-Network-Simulation
Designing a complex corporate network topology using Cisco Packet Tracer simulation tool.
The network follows the basic principles and several types of configurations that I have implemented. The network is divided into
a central part and a branch office, which are located far from each other, but are fully interconnected via an encrypted
IPSec VPN tunnel through two external ISPs. It also includes a DMZ server area and a
4G mobile network. This project demonstrates that even in a simulator like Packet Tracer, it is possible to create a fully
functional, secure, and professionally designed network topology that connects the central and
remote locations of a company.

I am also publishing a file related to the login details in the topology in this repository. All the data is there along with the names of the departments and which department/part of the network the data belongs to. Since this network topology is not fully adapted for English and uses equivalents in the original language, I decided to translate all the department names in this file so that it is known which department in English it actually is. These original names remained unchanged in the topology descriptions, because the login details start with the first letters of the names of these departments, which worked for me during the configuration. Since this is a school project, I had to work with my native language. I really hope that I made the topology a little more readable with this translation!
